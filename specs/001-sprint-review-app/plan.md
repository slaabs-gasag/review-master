# Implementation Plan: Sprint Review App

**Branch**: `001-sprint-review-app` | **Date**: 2026-05-22 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-sprint-review-app/spec.md`

## Summary

Build a local-only Nuxt 4 sprint review tool with three modes: Planning (agenda editor with
drag-to-reorder), Presenter (full-screen slideshow with keyboard navigation and Q&A capture),
and Archive (completed reviews with re-watch support). Data persists in SQLite via Nuxt server
API routes; screenshots stored as files in `.data/screenshots/`. UI follows the LaabsLab dark
design system from the Claude Design prototype, implemented with `@nuxt/ui` v4 components.

## Technical Context

**Language/Version**: TypeScript 6, Vue 3, Nuxt 4.4

**Primary Dependencies**: `@nuxt/ui` v4 (TailwindCSS v4), `better-sqlite3` v12 (already installed)

**Storage**: SQLite at `.data/review-master.sqlite`; screenshot files at `.data/screenshots/`

**Testing**: Vitest (unit + Nuxt component), Playwright (e2e, Chromium)

**Target Platform**: Local browser (Chromium), desktop viewport only

**Project Type**: Web app (local-only, no deployment)

**Performance Goals**: Sub-2s cold start; instant slide transitions in presenter mode

**Constraints**: Local-only, no auth, no cloud deps, no mobile/touch support required

**Scale/Scope**: Single user, up to ~30 items per review, no multi-team auth isolation
